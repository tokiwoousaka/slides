Lensの仕組み

Lensのアクセッサ

関数の型をtype宣言しただけ

type Lens s t a b
= forall f. Functor f => (a -> f b) -> s -> f t

Lensのアクセッサ

  • a : sが内包する型
  • b : tが内包する型
  • s : 操作したい元の型
  • t : 値の書きかえ後の型

type Lens s t a b
= forall f. Functor f => (a -> f b) -> s -> f t

Lensのアクセッサ

s と t の型が違う理由、
タプルの例がわかりやすいんです

type Lens (Int, x) (String, x) Int String
= forall f. Functor f =>
(Int -> f String) -> (Int, x) -> f (String, x)

Lensのアクセッサ

f に入るのは Identity または Const
どっちもFunctor、ねこ向けなので詳細は割愛

Identity a = Identity a
Const a b = Const a

Lensのアクセッサ

値の書き替え
Identityの部分は無視して読めるので、ただのmap関数

type Lens (Int, x) (String, x) Int String
= (Int -> Identity String) -> (Int, x) -> Identity (String, x)

Lensのアクセッサ

値の取り出し
Constの第二引数は無視できる事に注目

type Lens (Int, x) (Int, x) Int String
= (Int -> Const Int Int) -> (Int, x) -> Const Int (Int, x)

Lensのアクセッサ

Functorの制約が必要な理由
型あわせのため

_1 :: Lens (a, v) (b, v) a b
_1 f (x, y) = fmap (\x' -> (x', y)) (f x)

Lensの限界とPrism

直和型が混ざるとLensだけではキツい

-- Getしたい
print $ case foo^._2 of
Left inner -> Just (inner^._1)
Right _ -> Nothing

-- Setしたい
(_2%~(\case
Left x -> Left $ x&_1.~111
Right x -> Right x)) foo

Lensの限界とPrism

Prismならこう書ける

let rightHoge = Right "Hoge" :: Either Int String

-- Setする時、マッチしない場合は何もしない
rightHoge&_Right.~"Piyo" -- Right "Piyo"
rightHoge&_Left.~999 -- Right "Hoge"

-- Getの場合、結果はMaybe型なので演算子が変わる
rightHoge^?_Left -- Nothing
rightHoge^?_Right -- Just "Hoge"

Lensの限界とPrism

ただし、取得したい値がMonoidの場合に限り
Lensと同じ演算子でGetできる(マッチしなかったらmempty)

Just "hoge"^._Just -- OK "hoge"
Just 114514^._Just -- NG
Just (Sum 184)^._Just -- OK Sum { getSum = 184 }
(Nothing :: Maybe (Sum Int))^._Just -- OK Sum { getSum = 0 }

Lensの限界とPrism

Lensと合成した場合、全体はPrismになる

let foo = (1, Left (999, 3), 4)
:: (Int, Either (Int, Int) String, Int)
let bar = (1, Right "Test", 4)
:: (Int, Either (Int, Int) String, Int)

print $ foo^?_2._Left._1 -- Just 999
print $ foo&_2._Left._1.~111 -- (1,Left (111,3),4)
print $ bar^?_2._Right -- Just "Test"
print $ bar^._2._Rightn -- "Test"

Prismの真の姿

Prismの実際の定義と
さっきお見せした定義を比較

-- さっきの
type Prism s t a b
= forall f. Applicative f => (a -> f b) -> s -> f t

-- 実際
type Prism s t a b
= forall p f. (Choice p, Applicative f) => p a (f b) -> p s (f t)
